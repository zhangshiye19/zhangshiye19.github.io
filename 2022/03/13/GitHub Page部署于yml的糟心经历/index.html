Redis在SpringBoot上的集成

发表于2022-03-13|更新于2022-03-13
|阅读量:

这是一次及其糟心的部署经历，本来以为GitHub Page部署起来应该也不麻烦，但没想到倒在了yml上。在配置butterfly主题文件中，butterfly的默认每篇文章都需要图片，而我为了让界面更好看而且更省事一点，查阅到，配置文件有个default_cover属性，在文章默认没有配置封面的时候会自动使用该属性下的图片。但配置的过程中这个属性无论如何都不生效，首先我确定这个配置文件是有效的，因为我修改的其他内容都生效了，就只有default_cover始终都无效，对此我对该属性产生了怀疑：有没有可能default_cover属性代表的是其他某种未知的设置，也就是封面配置属性不是它，而经过一下午的百度,bing结果却都指向了default_cover没有问题。接着，我虽然没找到封面设置结果，却找到了default_top_img，这是文章另一组需要设置的图片，而这个是设置完成之后立马起效。而就在我为default_top_img调整缩进的时候我突然注意到，yml有非常严格的缩进要求，回头一看，果然，default_cover虽然值都有，但是多了两空格缩进…修复缩进，重启，生效…这就离谱

创建GitHub仓库

github创建一个名为${username}.github.io的仓库，点进仓库的设置，选择github pages完成配置

安装 hexo 环境

  1. npm install hexo-cli
  2. hexo init myblog
  3. cd myblog
  4. npm install hexo-deployer-git hexo-theme-butterfly

    配置主题以及基本设置

_config.yml

配置theme: butterfly没啥好说的，其他看着随便配吧

_config.butterfly.yml

在根目录下创建_config.buuterfly.yml，默认情况下，如果通过git下载的主题文件夹下会有个_config.yml配置文件，但我是npm下载的，所以就算有肯定也在node_module下面，这时候可以直接在根目录创建这个文件，这个文件同样也会生效

cover

新建page时默认的封面，默认下每个页面都需要，不需要的话可以关掉，如果没有设置会从default_cover随即找一张

default_top_img

同上，不过这个是缩略图
